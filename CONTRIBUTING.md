## Quick Summary

1.  **Issue First**: Please open an issue to discuss significant changes before starting work.
2.  **Layers**: Respect the project's layered architecture (Controller -> Service -> Repository).
3.  **i18n**: Never add hard-coded strings. All user-facing text must be localized in English and Turkish.
4.  **Quality**: Ensure `bun check` passes before submitting your pull request.

Thank you for helping make Booking Calendar better for everyone!
